# 21st.dev Primitive Implementation Matrix

Last updated: 2026-06-07

This matrix records the app-native 21st.dev-inspired primitive swap for TJC Stock Media. It separates **source status** from **implementation status** so the documentation does not overclaim registry access.

Source status meanings:

- `source_retrieved`: registry/source code was downloaded and inspected in this repo or scratch evidence.
- `public_reference_only`: public reference page/category was available, but usable source is not diff-checkable here.
- `unavailable_deferred`: no usable source/reference was available or the primitive was intentionally deferred/rejected.

Implementation status meanings:

- `implemented-visible`: implemented in app-native code and visible in real DAM workflows/screenshots.
- `implemented-partial`: implemented for safe subset or support shell, with documented gaps.
- `deferred-safe`: not implemented because doing so safely needs production data/policy.
- `rejected`: intentionally not implemented.
- `not-started`: no meaningful implementation yet.

No wholesale shadcn setup was added. 21st.dev references were used as primitive/behavior references; components were rebuilt in the existing Next/Tailwind/Lucide system.

## Safety Summary

- ResourceSpace remains the source of truth for assets, metadata, workflow state, and approval status.
- Raw ResourceSpace status remains separate from computed portal reuse state in cards, detail, review, and badges.
- Pending review writes remain local/server-routed until ResourceSpace API write mapping exists.
- Unsafe downloads still return 403; Viewer cannot access original/master content.
- No client-side ResourceSpace secrets, media files, or fake successful ResourceSpace writes were introduced.
